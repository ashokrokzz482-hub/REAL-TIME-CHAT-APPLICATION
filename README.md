## DESCRIPTION

The Real-Time Chat Application is a modern web-based communication platform developed to enable users to exchange messages instantly over the internet. The application uses WebSockets to establish a persistent, two-way communication channel between the client and the server. Unlike traditional HTTP requests, which require a new connection for every interaction, WebSockets maintain a continuous connection, allowing messages to be sent and received immediately without refreshing the page. This technology provides a fast, seamless, and interactive chatting experience similar to popular messaging applications.

The front-end of the application was developed using React.js, a powerful JavaScript library for building dynamic and component-based user interfaces. React enables efficient rendering of UI components, making the application responsive and easy to maintain. The interface was designed with reusable components such as Join Room, Message History, Message Bubble, and Message Composer, providing a clean and organized code structure. Users can enter their username, join a chat room, send messages, and receive messages from other connected users in real time.

The back-end was built using Node.js with the Express.js framework. Express provides a lightweight and efficient environment for handling server-side operations and routing. To enable real-time communication, the project uses the Socket.IO library, which simplifies WebSocket implementation and automatically handles connection management, event handling, and message broadcasting between clients. When a user sends a message, the server immediately broadcasts it to every participant in the same chat room, ensuring that all users receive updates instantly.

The application also maintains message history, allowing users joining a room to view previous conversations instead of starting with an empty chat. This improves the usability of the application and creates a more realistic chat environment. The responsive user interface ensures that the application functions well on both desktop and mobile devices. Modern CSS techniques, including Flexbox, animations, hover effects, and glassmorphism styling, were used to create an attractive and user-friendly interface.

The project follows a modular folder structure, separating the front-end and back-end into different directories. This organization improves readability, simplifies debugging, and makes future enhancements easier. Reusable React components reduce code duplication and improve maintainability. During development, emphasis was placed on writing clean, organized, and understandable code using meaningful component names and proper project structure.
